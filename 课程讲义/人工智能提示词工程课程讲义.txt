课程名称：人工智能提示词工程（AI Prompt Engineering）

课程性质：实践导向的Agent开发入门课程

核心目标：掌握从提示词设计到AI Agent完整开发的全链路能力

预期成果（三选一考核）：

1. 开发Agent + 项目报告（满分100分）

2. 撰写课程讲义 + 项目报告（满分95分）

3. 大学生创新创业/职业生涯规划大赛材料 + 项目报告（满分90分）

课程成果要求：

1. 发布在Github上的项目

2. 发布在Github上的人工智能Agent开发入门教程讲义

3. 输出一份高质量的参加大学生竞赛的参赛文件，并且参加比赛获奖

二、核心概念体系

System Prompt：定义AI的身份、规则与边界，是全局生效的底层设定

User Prompt：用户根据即时需求输入的具体任务，是动态变化的临时指令

Token：词元，是大语言模型处理文本的基本单位

Context：上下文，包含历史对话记录等信息

Context Length：上下文长度，指模型能处理的最大文本长度

RAG：检索增强生成，通过检索外部知识库增强生成能力

Function Call：让LLM能够调用外部工具的机制

Skill：技能，可复用的功能模块

MCP：模型上下文协议

Agent：智能体，能够自主完成任务的AI系统

KV Cache：键值缓存，用于加速模型推理

MoE架构：混合专家架构，提升模型能力

量化(Q4/Q6)：模型参数量化，减少显存占用

第2章 提示词工程基础技巧

理论内容：

角色定义技巧：指定AI角色身份、描述用户角色身份

结构控制：双AI辩论规划回复结构、回答模板（案例）设计

提示词的基本结构：

角色（Role）：给模型分配身份，控制语气

任务（Task）：明确要完成的具体任务

输入（Input）：提供必要的输入信息

输出格式（Output Format）：指定输出的格式要求

约束（Constraint）：设置输出的限制条件

示例（Example）：提供参考示例

第3章 本地模型配置与基础实验

六、提示词工程核心技术总结

1. 零样本提示（Zero-Shot）：只给任务描述，不给示例，适合简单、通用任务

2. 少样本提示（Few-Shot）：给少量示例，引导模型学习模式，适合复杂任务

3. 角色提示（Role Prompting）：给模型分配身份，控制语气和行为

4. 链式思维（Chain of Thought）：引导模型分步思考，适合复杂推理或计算

5. 结构化输出（JSON）：指定输出格式，便于程序处理

6. 自我一致性（Self-Consistency）：多次生成取多数答案，提高准确性

7. ReAct（Reasoning + Acting）：结合推理与行动，适合需要多步操作的任务

8. 提示词模板化：将成熟提示词封装为模板，提高复用率

七、最佳实践与常见误区

最佳实践：

- 指令清晰明确，避免歧义

- 提供必要的上下文信息

- 指定输出格式和约束条件

- 使用示例引导模型行为

- 逐步优化提示词，通过测试迭代改进

常见误区：

- 指令过于模糊或冗长

- 忽略模型的参数配置

- 未考虑上下文长度限制

- 不进行测试和迭代优化

- 期望模型理解隐含需求
